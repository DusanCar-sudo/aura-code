①

User input

Task is entered as text: aura "refactor auth"

②

CLI — argument parsing

minimist parses flags, .aura.json is loaded, env variables are merged. Decision: REPL or single task?

③

Router

Decides whether the task goes directly into the agent loop or is orchestrated (decomposed into sub-tasks for specialists).

④

Model selection

selectModel() picks the best model from the same family based on episode history — which model has most often succeeded at similar tasks.

⑤

Provider Layer

Abstracts the API call: retry (exponential backoff + jitter), circuit breaker (after 5 failures), fallback to next model.

⑥

Ruby alternation

If Ruby is enabled (default), RubyAlternator assesses whether a free local Ollama model can solve the task — based on competence from past episodes. If it fails, escalates to the large model.

⑦

Agent loop

Main loop: LLM generates response → parses tools → safety check → execution → continue. Compactor summarizes history at >70% context.

⑧

Tool execution

The registry calls the right tool: read_file, edit_file, run_shell, web_search, etc. Each tool is an independent module.

⑨

Episode capture

Every outcome is saved as an Episode — task, model, success/failure, duration. Episodes are the single source of truth for "learning".

⑩

Verification (feedback loop)

If enabled, runs tests. Failure → retry. The dashed blue line in the diagram shows how information loops back to ③ (model selection learns from episodes).
